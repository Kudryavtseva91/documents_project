# Внедрение ServiceDesk на базе GLPI

**Роль:** Системный аналитик / Руководитель проекта  
**Период:** 2025-2026  
**Результат:** Оптимизация работы L1 и L2, сокращение времени обработки заявок на **40%**

---

## 📋 Описание проекта

Внедрение централизованной системы ServiceDesk для учёта, диспетчеризации и анализа обращений всех подразделений компании.

---

## 🎯 Цели проекта

- Создать единую точку входа для всех обращений
- Автоматизировать маршрутизацию заявок
- Внедрить SLA-механизмы для контроля качества
- Обеспечить прозрачность и аналитику процессов

---

## 📊 Результаты

| Метрика | До внедрения | После внедрения |
|---------|-------------|-----------------|
| Время обработки заявки | 4-6 часов | 1.5-2 часа |
| Ручная маршрутизация | 100% | 20% |
| Соблюдение SLA | 0% | 95% |
| Нагрузка на L1 | 100% | 40% |

---

## 🔧 Моя роль и задачи

### Анализ AS-IS
- Провела интервью со стейкхолдерами (4+ отделов)
- Проанализировала текущие процессы и боли
- Выявила узкие места в Redmine + почта

### Сбор и управление требованиями
- Собрала и приоритизировала требования (MoSCoW)
- Сформировала User Stories и Acceptance Criteria
- Создала матрицу требований

### Проектирование TO-BE
- Разработала BPMN-диаграммы процессов
- Спроектировала модель данных
- Разработала правила маршрутизации (последовательная/параллельная)
- Спроектировала SLA-механизмы с учётом рабочего времени

### Настройка и кастомизация
- Настроила GLPI под процессы компании
- Кастомизировала интерфейс (названия кнопок, поля)
- Добавила колонку "Группа инициатора" в таблицу заявок
- Настроила интеграции: почта, 1С, ADFS, SCCM

### Внедрение и обучение
- Провела обучение агентов L1 и L2
- Запустила пилотный проект
- Собрала обратную связь и доработала процессы
- Вывела в промышленную эксплуатацию

---

## 🛠️ Технологии

- **Платформа:** GLPI
- **Моделирование:** BPMN 2.0, UML
- **Интеграции:** REST API, Webhook, IMAP/SMTP
- **Безопасность:** ADFS (SSO), ролевая модель
- **Стандарты:** ITIL, SLA
