# Техническое задание на внедрение ServiceDesk

## 1. Цели и задачи проекта

### Бизнес-цели:
- Сокращение времени обработки заявок на 30-40%
- Повышение прозрачности процессов поддержки
- Внедрение SLA-механизмов
- Снижение нагрузки на L1 поддержку

### Задачи проекта:
1. Внедрить централизованную систему учёта обращений
2. Автоматизировать маршрутизацию заявок
3. Настроить SLA по отделам
4. Обеспечить интеграцию с существующими системами

---

## 2. Требования к системе

### Функциональные требования:

#### 2.1. Управление пользователями
- 6 ролей: инициатор, наблюдатель, исполнитель, руководитель, диспетчер, администратор
- SSO через ADFS
- Синхронизация из Active Directory

#### 2.2. Автоматическое создание заявок
- Из писем на support@company.ru (каждые 2 минуты)
- Извлечение темы, тела, вложений
- Определение инициатора по email

#### 2.3. Маршрутизация
- Правила: Тип заявки → Группа исполнителей
- Последовательное назначение
- Параллельное назначение
- Возврат на 1 линию с комментарием

#### 2.4. Статусы и жизненный цикл
- Новая → В распределении → В работе → В ожидании → Выполнена
- Отклонена, Дубликат
- Цветовая индикация статусов

#### 2.5. SLA и метрики
- 5 типов времени: реакция, диспетчеризация, реакция исполнителя, время в работе, время жизни
- Остановка в нерабочее время
- Уведомления при 70% истечения SLA
- Эскалация руководителю

#### 2.6. Оценка качества
- ХОРОШО, ПЛОХО, ВЕРНУТЬ В РАБОТУ
- 24 часа на оценку
- Автозакрытие без оценки

#### 2.7. Интеграции
- Почта (IMAP/SMTP)
- 1С (структура подразделений, графики работы)
- ADFS (SSO)
- SCCM (автоустановка ПО)
- REST API, Webhook, Чат-боты

#### 2.8. Кастомизация без кода
- Настройка типов заявок, полей, приоритетов
- Правила маршрутизации через UI администратора

---

### Нефункциональные требования:

#### Производительность
- Время отклика < 2 сек (95% запросов)
- Загрузка 1000 заявок < 3 сек
- Пропускная способность: 750 заявок/день
- Поддержка до 500K заявок в БД

#### Безопасность
- ADFS (SSO)
- 6 ролей, принцип минимальных привилегий
- HTTPS, маскирование ПД в логах
- Аудит входов, хранение логов 1 год
- Соответствие 152-ФЗ

#### Доступность
- 99.5% в рабочее время (8:00–22:00)
- RTO < 4 часа
- Ежедневный бэкап БД

## 3. Требования к интеграциям

### 3.1. Интеграция с почтой
- Протоколы: IMAP, SMTP
- Автоматическое создание заявок из писем
- Прикрепление вложений к заявке
- Отправка уведомлений по email

### 3.2. Интеграция с ADFS
- Протокол: SAML 2.0
- Автоматическая авторизация пользователей
- Синхронизация ролей и прав

### 3.3. Интеграция с 1С
- Синхронизация структуры подразделений
- Графики работы сотрудников
- Список пользователей
