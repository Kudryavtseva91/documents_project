# Техническое задание на AI-ботов поддержки

## 1. Цели проекта

- Снизить нагрузку на агентов L1 на 60%
- Ускорить обработку типовых запросов
- Автоматизировать классификацию заявок
- Повысить качество поддержки

---

## 2. Состав ботов

### 2.1. AI-бот поддержки (RAG-based)

**Задача:** Отвечает на вопросы пользователей на основе базы знаний

**Функциональность:**
- Обработка текстовых запросов на естественном языке
- Поиск ответов в базе знаний (Confluence, B24)
- Генерация ответа с помощью LLM
- Оценка уверенности ответа

**Технологии:**
- RAG (Retrieval-Augmented Generation)
- Embeddings (векторизация текстов)
- LLM (ChatGPT API / YandexGPT)
- Confluence API для доступа к базе знаний

**Сценарии:**
1. Пользователь задаёт вопрос
2. Бот ищет релевантные статьи в базе знаний
3. Формирует ответ на основе найденных статей
4. Если уверенность низкая — передаёт агенту

---

### 2.2. AI-бот L1 (классификатор)

**Задача:** Классифицирует заявки, назначает категорию и приоритет

**Функциональность:**
- Анализ текста заявки
- Определение типа (инцидент, запрос, проблема)
- Назначение категории
- Определение приоритета (Critical, High, Medium, Low)
- Автоматическое создание заявки в ServiceDesk

**Технологии:**
- NLP (обработка естественного языка)
- ML-классификация
- GLPI API (создание заявок)

**Сценарии:**
1. Заявка поступает в систему
2. Бот анализирует текст
3. Определяет тип, категорию, приоритет
4. Создаёт заявку с заполненными полями

---

### 2.3. AI-бот аналитики

**Задача:** Строит отчёты и дашборды по запросу

**Функциональность:**
- Обработка запросов на естественном языке
- Выполнение SQL-запросов к БД
- Визуализация данных
- Формирование отчётов в PDF/Excel

**Технологии:**
- SQL
- Redash API
- NLP для интерпретации запросов

**Сценарии:**
1. Пользователь запрашивает отчёт
2. Бот интерпретирует запрос
3. Формирует SQL-запрос
4. Выполняет и визуализирует результат

---

### 2.4. AI-бот транскрибатор

**Задача:** Расшифровка аудио/видео обращений

**Функциональность:**
- Загрузка аудио/видео файлов
- Распознавание речи (Speech-to-Text)
- Создание текстовой расшифровки
- Извлечение ключевых моментов

**Технологии:**
- Whisper API
- Speech-to-Text
- NLP для извлечения сути

**Сценарии:**
1. Пользователь загружает аудиозапись
2. Бот распознаёт речь
3. Создаёт текстовую расшифровку
4. Извлекает ключевые моменты

---

## 3. Архитектура

### Общая схема
Пользователь → Чат-бот → API Gateway → AI-боты → ServiceDesk
↓
База знаний

### Взаимодействие с ServiceDesk
AI-бот L1 → GLPI API → Создание заявки
↓
Классификация

---

## 4. Требования к данным

### База знаний
- Формат: Confluence, Bitrix24
- Структура: иерархическая (разделы → статьи)
- Обновление: ежедневное

### Обучение ботов
- Датасет: история заявок (10000+)
- Разметка: тип, категория, приоритет
- Метрики: точность > 90%

---

## 5. Критерии приёмки

- [ ] Точность классификации > 90%
- [ ] Время ответа < 2 сек
- [ ] Интеграция с ServiceDesk работает
- [ ] База знаний подключена
- [ ] Логирование и мониторинг настроены
- [ ] Инструкции для пользователей готовы
